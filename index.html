Key concepts

What I can explain from hands-on experience

Terraform state
Terraform's memory of what it created. Maps .tf resources to real IDs. Stored in S3 with DynamoDB locking. Drift = gap between state and reality — proved by deleting a resource manually and watching plan detect it.
VPC 3-tier design
Public subnet: ALB + NAT Gateway. Private subnet: app servers, no public IP. Data subnet: databases, zero internet route. Security groups use security_groups references not CIDR so auto-scaling doesn't break rules.
Lambda cold start
Proved with real data: container_age = 0.0s cold, 3.85s warm, 7.59s still warm. Code outside handler() runs once. Fix: Provisioned Concurrency for payment flows where cold starts are unacceptable.
Kubernetes self-healing
Deleted a pod. ReplicaSet Controller detected desired=3, actual=2 in etcd. New pod appeared in 41 seconds. Service routed traffic immediately — user never noticed. Proved not described.
K8s Secrets not encrypted
Decoded SuperSecretBankPassword123 from a Secret in one line — no special tools. Base64 is encoding not encryption. Fix: etcd encryption-at-rest + External Secrets Operator backed by real secrets manager.
NetworkPolicy gap
Wrote a correct NetworkPolicy restricting a namespace. Traffic from blocked namespace still got through. Root cause: kindnet does not enforce NetworkPolicy. Calico does. Tested not assumed — documented in README.
Secrets Manager pattern
Store full connection JSON: host, port, username, password, dbname. App reads one secret at startup. Lambda execution role gets secretsmanager:GetSecretValue on that specific ARN only — nothing else.
OIDC vs static credentials
GitHub Actions proves identity to AWS cryptographically. Gets 15-minute token. Nothing stored. If GitHub is breached — no credentials to steal. Proved with real ASIA prefix STS output showing 1-hour expiry.
